# Production Snapshot Placeholder

This folder is a local placeholder for asynchronously refreshed production KB snapshots.

In a real deployment, domain teams should publish approved snapshot documents here through ingestion jobs or pipelines instead of editing app code directly.

Suggested flow:
- strategy team publishes portfolio priorities and OKR extracts
- data team publishes maturity, quality, and lineage snapshots
- technology team publishes service health, architecture, and delivery-capacity snapshots
- market team publishes macro and micro trend digests
- governance team publishes policy and control snapshots
